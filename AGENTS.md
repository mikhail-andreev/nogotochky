## Правила работы
- Работать с backend из `apps/backend`.
- Для dev использовать Docker Compose.
- Не менять структуру каталогов без согласования.
- Не перезаписывать данные пользователя и не удалять файлы без запроса.
- При изменениях в коде обновлять `docs/PRD.md` и `docs/SDD.md`, чтобы документация соответствовала реализации.
- После изменения кода  запускать сервисы через  Docker Compose и проверять с помщью curl что они отвечают по http
- git push делаем только по прямому указанию
